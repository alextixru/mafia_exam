# mafia-opros — контекст проекта

Discord-бот для проведения опросов внутри сервера. Главное сообщение с
выпадающим списком опросов; при выборе опроса бот ведёт пользователя через
вопросы в эфемерных сообщениях, заменяя одно сообщение на следующее.

## Стек

- **Runtime:** Bun
- **Язык:** TypeScript
- **Discord:** discord.js
- **Хранилище опросов:** JSON-файлы в `polls/`
- **Хранилище сессий:** in-memory `Map` + дамп в `data/sessions.json` после
  каждого ответа (переживает рестарт)
- **Отчёты:** при завершении опроса отчёт улетает сообщением в чат
  (json/embed). За пределами одной сессии прохождения ничего не храним
- **Админка:** отдельный Bun-сервер с простым HTML для редактирования
  `polls/*.json` (на потом, в MVP правим руками)

## Поведение

### Главное сообщение
Persistent embed с String Select Menu. В меню — список доступных опросов.
Не удаляется, висит в канале постоянно. При рестарте бот находит/создаёт
это сообщение по сохранённому id.

### Выбор опроса
Юзер кликает в Select Menu → бот отвечает ephemeral с первым вопросом.
Если у юзера уже есть незавершённая сессия по этому опросу — продолжает
с того вопроса, где остановился.

### Типы вопросов

1. **`free`** — свободный текст. В эфемерке — кнопка «Ответить» →
   открывается Modal с TextInput. Ограничения Discord: заголовок Modal
   до 45 символов, поэтому полный текст вопроса остаётся в embed под
   модалкой; в Modal — только короткий title (например, «Ваш ответ»).
2. **`single`** — выбор одного из вариантов. String Select Menu с
   `min_values=1, max_values=1`. Discord не поддерживает `defaultValues`
   для строкового select без особых флагов, поэтому при возврате к
   вопросу текущий ответ показывается в тексте embed
   («Ваш текущий ответ: X»), а не подсветкой опции.
3. **`multi`** — выбор нескольких. String Select Menu с `min_values≥1,
   max_values=N`.

### Навигация
- Кнопка **«Назад»** — вернуться к предыдущему вопросу и отредактировать
  ответ.
- Кнопка **«Далее»** — перейти к следующему (для `single`/`multi`
  отображается после первого выбора; для `free` ответ подтверждается
  отправкой Modal).
- При замене вопроса предыдущая эфемерка редактируется (не отправляется
  новая). Это важно: 15-минутный лимит токена интеракции — каждый
  компонент использует свой токен, поэтому время «жизни» рассчитываем
  от последнего взаимодействия.

### Завершение
- На последнем вопросе кнопка **«Завершить»** → отправляем отчёт
  сообщением в основной канал (или в указанный канал отчётов), сессию
  чистим из памяти и из `sessions.json`.

## Подводные камни (зафиксированы)

1. **Эфемерные сообщения теряются** при перезапуске клиента Discord,
   переключении устройства, нажатии «Dismiss message», истечении
   ~15 минут. Поэтому прогресс хранится на сервере, и при повторном
   выборе опроса в Select Menu бот восстанавливает состояние.
2. **Modal под эфемеркой** — текст самого вопроса остаётся в embed,
   Modal показывает только короткий title (≤45 симв) и поля ввода.
3. **`defaultValues` в String Select** — нет полноценного API,
   подсветка прежнего выбора делается текстом в embed.
4. **Persistent главное сообщение** — id сообщения сохраняем в
   `data/state.json`, при старте бот его проверяет и при необходимости
   пересоздаёт.

## Структура (планируемая)

```
mafia-opros/
├── CONTEXT.md            ← этот файл
├── package.json
├── tsconfig.json
├── .env                  ← DISCORD_TOKEN, GUILD_ID, CHANNEL_ID
├── .env.example
├── .gitignore
├── polls/                ← JSON-определения опросов
│   └── example.json
├── data/                 ← runtime-данные (gitignored)
│   ├── sessions.json
│   └── state.json
└── src/
    ├── index.ts          ← entry point, инициализация клиента
    ├── config.ts         ← чтение env
    ├── types.ts          ← типы Poll, Question, Session
    ├── polls.ts          ← загрузка polls/*.json
    ├── sessions.ts       ← in-memory + json-persist
    ├── ui/
    │   ├── main-menu.ts  ← главный embed + Select Menu опросов
    │   ├── question.ts   ← рендер вопроса (free/single/multi)
    │   └── report.ts     ← финальный отчёт
    └── handlers/
        ├── select.ts     ← обработчик Select Menu
        ├── button.ts     ← обработчики кнопок (Назад/Далее/Ответить/Завершить)
        └── modal.ts      ← обработчик Modal-submit для free-вопросов
```

## Формат опроса (черновик)

```json
{
  "id": "onboarding",
  "title": "Знакомство",
  "description": "Расскажите о себе",
  "questions": [
    { "id": "name", "type": "free", "text": "Как вас зовут?" },
    { "id": "role", "type": "single", "text": "Ваша роль?",
      "options": [
        { "value": "player", "label": "Игрок" },
        { "value": "leader", "label": "Ведущий" }
      ]
    },
    { "id": "exp", "type": "multi", "text": "Опыт в каких ролях есть?",
      "options": [
        { "value": "mafia", "label": "Мафия" },
        { "value": "sheriff", "label": "Шериф" },
        { "value": "doctor", "label": "Доктор" }
      ],
      "min": 1, "max": 3
    }
  ]
}
```

## Окружение

- `DISCORD_TOKEN` — токен бота
- `GUILD_ID` — id сервера (для регистрации slash-команд только на нём,
  быстрее глобальной регистрации)
- `MAIN_CHANNEL_ID` — канал, где висит главный embed
- `REPORT_CHANNEL_ID` — куда падают отчёты (можно совпадать с main)
